01

토큰이란? 누적 소비 구조 이해

Claude의 처리 방식과 토큰이 업무에 미치는 영향

⚡ Strategic Insight — 98.5% vs 1.5% 법칙

메시지가 100개 이상 누적될 경우, 전체 토큰의 98.5%가 이전 대화를 다시 읽는 데 소모됩니다. 실제 새로운 작업에 투입되는 비중은 단 1.5%에 불과합니다.

🔄

누적 대화 읽기 구조

Claude는 매 응답 시 이전 대화 기록 전체를 다시 스캔합니다. 대화가 길어질수록 '과거 복기'에 대부분의 자원이 투입됩니다.

📊

토큰 = 추론 용량

토큰은 단순 비용 단위가 아니라 AI의 집중력과 업무 연속성을 결정짓는 핵심 자원입니다.

⚠️

맥락 왜곡 위험

누적 토큰이 임계치를 넘으면 지침을 망각하거나 환각(Hallucination) 현상이 발생할 확률이 높아집니다.

02

파일 포맷 전략: 마크다운을 써야 하는 이유

포맷별 토큰 점유율 비교 및 전처리 의무 규칙

💡 핵심 규칙

모든 외부 문서(PDF, Word 등)는 업로드 전 반드시 마크다운(.md)으로 변환하여 입력하세요. 이미지 처리로 인한 토큰 폭증을 방지하는 핵심 전략입니다.

Markdown 권장 10 (기준)
PDF 70% 추가 소모 33
HTML 90% 추가 소모 100

03

수정 기능 우선 원칙

후속 메시지 대신 연필 아이콘으로 프롬프트 보정하기

❌

비효율 방식 — 후속 메시지

결과가 마음에 안 든다고 새 메시지를 보내면 오답 + 재질문이 모두 컨텍스트에 누적됩니다. 토큰 낭비가 2배로 발생합니다.

✏️

효율 방식 — 연필 아이콘(수정)

메시지 옆 연필 아이콘을 클릭하여 프롬프트를 직접 수정하면, 오답 기록이 삭제되고 새로운 경로로 재추론합니다. 동일 결과를 훨씬 적은 토큰으로 달성할 수 있습니다.

💡 토큰 아비트리지(Token Arbitrage)

수정 기능은 오답 기록을 완전히 제거하고 새 경로로 재시작하기 때문에, 후속 메시지 방식 대비 컨텍스트 오염 없이 동일한 목표를 달성합니다.

04

20-메시지 세션 로테이션

컨텍스트 이관으로 맥락은 유지하고 토큰은 초기화

⚠️ 임계치 경보: 단일 세션 내 메시지가 15~20개를 초과하면 즉시 새 세션을 시작하세요. 초과 시 토큰의 98.5%가 과거 복기에 소모됩니다.
1

핵심 맥락 요약 요청

세션 종료 직전, Claude에게 다음과 같이 요청하세요:
"현재까지의 핵심 맥락을 새 세션용 마크다운 요약본으로 작성해 줘"

2

새 세션 시작

현재 세션을 종료하고 새 대화창을 엽니다. 이전 토큰 누적이 완전히 초기화됩니다.

3

요약본을 첫 메시지로 입력

마크다운 요약본을 새 세션의 첫 메시지로 붙여넣으면, 맥락은 유지하면서 토큰은 완전히 초기화된 상태에서 업무를 재개할 수 있습니다.

05

일괄 요청 (Batching) 전략

워크플로우를 하나의 프롬프트로 통합하여 효율 극대화

❌

비효율 — 단발성 질문

"이거 요약해줘" → 답변 → "이걸 분석해줘" → 답변 → "보고서 써줘"

매번 컨텍스트를 로딩하며 토큰이 3배로 소모됩니다.

✅

효율 — 일괄 요청 (Batching)

"다음 문서를 ① 요약하고 ② 주요 시사점을 분석한 뒤 ③ 경영진용 1페이지 보고서로 작성해 줘"

한 번의 컨텍스트 로딩으로 동일 작업 완료.

🎯 핵심 원칙

반복적인 컨텍스트 로딩 횟수를 줄이는 것이 핵심입니다. 요약 → 분석 → 리포트 등 연관 작업은 하나의 프롬프트에 통합하여 요청하세요.

07

프로젝트 기능 & 설정 활용법

반복 작업을 자동화하여 매번 입력하는 수고 없애기

A

Project Knowledge 활용

반복 참조하는 문서(회사 소개, 제품 스펙, 업무 규정 등)는 매 채팅마다 첨부하는 대신 '프로젝트' 기능에 공유 리소스로 등록하세요. 자동으로 컨텍스트에 포함됩니다.

B

System Instructions (맞춤 설정)

반복적인 페르소나·출력 형식·언어 스타일 지시는 매번 입력하지 말고 '개인 맞춤 설정' 또는 '프로젝트 지침'에 등록하세요. 시스템 프롬프트로 처리되어 사용자 메시지 토큰을 절감합니다.

C

불필요한 기능 비활성화

업무와 무관한 적응형 사고(Adaptive Thinking), 웹 검색(Web Search) 기능은 리소스를 과다 점유합니다. 기본 OFF 상태를 유지하고 필요 시에만 활성화하세요.

Claude Usage Tracker 설치 & 사용법

토큰 잔량을 실시간으로 파악하는 크롬 확장 프로그램

🔍 What is Claude Usage Tracker?

claude.ai에서 현재 내 토큰 사용량과 남은 한도를 실시간으로 보여주는 오픈소스 크롬 확장 프로그램입니다. 파일·프로젝트·채팅 히스토리별 토큰 소모량을 세분화하여 표시하고, 한도가 리필되면 알림까지 보내줍니다.

STEP 1 · 설치 크롬 웹 스토어에서 바로 설치
1

크롬 웹 스토어 접속

Chrome 브라우저 주소창에 아래 주소를 붙여넣거나, Chrome 웹 스토어에서 "Claude Usage Tracker"를 검색합니다.

https://chromewebstore.google.com/detail/claude-usage-tracker/knemcdpkggnbhpoaaagmjiigenifejfo 🔗 스토어 바로가기
2

"Chrome에 추가" 클릭

스토어 페이지에서 파란색 "Chrome에 추가" 버튼을 클릭합니다. 팝업이 뜨면 "확장 프로그램 추가"를 눌러 설치를 완료합니다.

3

툴바에 핀(고정) 설정

Chrome 우측 상단 퍼즐 조각 아이콘(🧩) → "Claude Usage Tracker" 옆 핀 아이콘 클릭 → 툴바에 항상 표시되도록 고정합니다.

STEP 2 · 확인 추적되는 토큰 항목과 화면 읽는 법
📁

Files (파일)

채팅에 업로드한 문서, Google Drive·GitHub 연동 파일의 토큰 소모량을 항목별로 표시합니다.

🗂️

Projects (프로젝트)

프로젝트 Knowledge 파일과 커스텀 지침(System Instructions)이 차지하는 토큰을 따로 확인합니다.

💬

Message History (대화 이력)

현재 세션의 전체 대화 컨텍스트, 시스템 프롬프트, 활성화된 도구(분석·아티팩트)의 토큰을 합산합니다.

📊 화면 색상 코드 읽는 법
초록 (70% 미만) — 여유 있음
노랑 (70~90%) — 주의 필요
빨강 (90% 초과) — 세션 전환 권고
STEP 3 · 활용 스마트하게 활용하는 방법
A

한도 리필 알림 받기

사용량이 한도에 근접했다가 리필될 때 자동 브라우저 알림이 발송됩니다. 별도 확인 없이 재개 시점을 바로 알 수 있습니다.

B

툴팁으로 상세 정보 확인

채팅 UI 내 길이(Length), 비용(Cost), 예상(Estimate), 캐싱 상태 요소에 마우스를 올리면 툴팁으로 상세 설명이 나타납니다.

C

20-메시지 로테이션과 연동

트래커가 노란색(70%)으로 전환되면 세션 로테이션을 준비하세요. 빨간색(90%) 도달 전에 새 세션으로 전환하면 업무 중단 없이 최적 상태를 유지할 수 있습니다.

⚠️ 주의사항: 웹 검색 결과와 Research(딥 리서치) 항목은 백엔드에서 처리되므로 정확한 추적이 어렵습니다. 또한 Claude Max 플랜의 경우 일부 버전에서 표시 오차가 발생할 수 있으니 참고 지표로 활용하세요.
